## 一、实验目标

本实验主要完成 VirtualBox 虚拟机环境的搭建，安装 Ubuntu Server 系统，并通过模板机克隆出多台虚拟服务器节点。通过本实验，理解宿主机、虚拟机、NAT、Host-Only 网络、静态 IP、主机名和 SSH 远程登录等基础概念。

## 二、核心概念

### 1. 宿主机

宿主机是运行虚拟机软件的真实电脑。本实验中，Windows 系统所在的电脑就是宿主机。

宿主机负责提供 CPU、内存、硬盘和网络资源，VirtualBox 则在宿主机上创建和管理虚拟机。

### 2. 虚拟机

虚拟机是在真实电脑中模拟出来的一台独立计算机。它拥有自己的虚拟硬盘、内存、网卡和操作系统。

本实验使用 VirtualBox 创建 Ubuntu Server 虚拟机，用来模拟真实服务器环境。

### 3. Ubuntu Server

Ubuntu Server 是 Ubuntu 的服务器版本，主要通过命令行进行操作。它更接近真实服务器环境，适合用于学习 Linux 操作、SSH 远程登录和网站部署。

### 4. NAT 网络

NAT 网卡用于让虚拟机通过宿主机访问外网。

在本实验中，NAT 网卡对应 Ubuntu 中的 enp0s3，获取到的地址为 10.0.2.15，用于执行 apt update、ping 外网等操作。

### 5. Host-Only 网络

Host-Only 网卡用于宿主机和虚拟机之间通信，也可以让多台虚拟机在同一个私有网段内互相访问。

在本实验中，Host-Only 网段为 192.168.56.0/24，虚拟机通过该网络实现 SSH 登录和节点互通。

### 6. 静态 IP

静态 IP 是手动指定并固定的 IP 地址。相比 DHCP 自动分配，静态 IP 更稳定，适合服务器环境。

本实验中配置的地址如下：

    ubuntu-template  192.168.56.101
    node1            192.168.56.102
    node2            192.168.56.103
    node3            192.168.56.104

### 7. SSH 远程登录

SSH 是一种安全的远程登录协议。通过 SSH，可以在 Windows PowerShell 中远程登录 Ubuntu 虚拟机。

例如：

    ssh dev@192.168.56.102

该命令表示使用 dev 用户登录 IP 地址为 192.168.56.102 的 Ubuntu 虚拟机。

## 三、实验环境

- 宿主机系统：Windows 11
- 虚拟机软件：Oracle VirtualBox
- 虚拟机系统：Ubuntu Server 22.04 LTS
- 虚拟机模板：ubuntu-template
- 用户名：dev
- 虚拟机网络：
  - 网卡 1：NAT
  - 网卡 2：Host-Only Adapter
- Host-Only 网段：192.168.56.0/24

## 四、实验过程

### 1. 创建 Ubuntu Server 模板机

首先在 VirtualBox 中创建一台名为 ubuntu-template 的虚拟机，配置 CPU、内存、虚拟硬盘，并挂载 Ubuntu Server ISO 镜像。

虚拟机基础配置如下：

    名称：ubuntu-template
    内存：2048 MB
    CPU：2 核
    硬盘：25 GB
    系统：Ubuntu Server 22.04 LTS

### 2. 配置虚拟机网络

为虚拟机配置两张网卡：

    网卡 1：NAT
    网卡 2：Host-Only Adapter

其中 NAT 用于访问外网，Host-Only 用于 Windows 宿主机与 Ubuntu 虚拟机通信。

### 3. 安装 Ubuntu Server

启动虚拟机后进入 Ubuntu Server 安装界面，选择默认 Ubuntu Server 安装类型，使用整块虚拟硬盘安装系统。

安装过程中创建 dev 用户，并勾选安装 OpenSSH Server，使虚拟机后续可以通过 SSH 远程登录。

### 4. 配置模板机静态 IP

安装完成后，登录 Ubuntu Server，查看网络接口：

    ip a

其中 enp0s3 是 NAT 网卡，enp0s8 是 Host-Only 网卡。

编辑 netplan 配置文件：

    sudo nano /etc/netplan/50-cloud-init.yaml

将 enp0s8 配置为静态 IP：

    enp0s8:
      dhcp4: false
      addresses:
        - 192.168.56.101/24

应用配置：

    sudo netplan try

配置完成后，模板机的 Host-Only IP 固定为 192.168.56.101。

### 5. 禁用 cloud-init 网络接管

由于 Ubuntu Server 的网络配置文件可能由 cloud-init 自动生成，为了避免重启后配置被覆盖，创建配置文件禁用 cloud-init 网络管理：

    sudo bash -c 'echo "network: {config: disabled}" > /etc/cloud/cloud.cfg.d/99-disable-network-config.cfg'

验证文件内容：

    cat /etc/cloud/cloud.cfg.d/99-disable-network-config.cfg

输出结果为：

    network: {config: disabled}

### 6. 配置 /etc/hosts

编辑 hosts 文件：

    sudo nano /etc/hosts

添加节点名称和 IP 的对应关系：

    192.168.56.101 template
    192.168.56.102 node1
    192.168.56.103 node2
    192.168.56.104 node3

这样可以通过主机名访问不同节点，例如：

    ping node1
    ping node2
    ping node3

### 7. 从模板机克隆 node1、node2、node3

关闭 ubuntu-template 后，使用 VirtualBox 克隆功能创建 node1、node2 和 node3。

每台克隆机都需要重新生成 MAC 地址，避免多台虚拟机网卡身份冲突。

克隆完成后，分别修改主机名和静态 IP：

    node1: 192.168.56.102
    node2: 192.168.56.103
    node3: 192.168.56.104

修改主机名示例：

    sudo hostnamectl set-hostname node1

修改 IP 后执行：

    sudo netplan apply

### 8. 重新生成 SSH Host Key

由于 node1、node2、node3 都是从模板机克隆出来的，它们最初会拥有相同的 SSH 服务器身份钥匙。

为了让每台虚拟机都有独立身份，分别执行：

    sudo rm -f /etc/ssh/ssh_host_*
    sudo dpkg-reconfigure openssh-server
    sudo systemctl restart ssh

然后在 Windows PowerShell 中清除旧的 known_hosts 记录：

    ssh-keygen -R 192.168.56.102
    ssh-keygen -R 192.168.56.103
    ssh-keygen -R 192.168.56.104

再重新通过 SSH 连接各节点。

### 9. 验证多机连通性

在 Windows PowerShell 中测试三台虚拟机是否可达：

    Test-Connection 192.168.56.102 -Count 3
    Test-Connection 192.168.56.103 -Count 3
    Test-Connection 192.168.56.104 -Count 3

在 node1 中测试与 node2、node3 的连通性：

    ping -c 3 node2
    ping -c 3 node3

同时测试外网连通性：

    ping -c 3 8.8.8.8

## 五、实验结果

本实验成功完成了 Ubuntu Server 虚拟机环境搭建，并通过模板机克隆出 node1、node2、node3 三台虚拟节点。

最终节点配置如下：

    ubuntu-template  192.168.56.101
    node1            192.168.56.102
    node2            192.168.56.103
    node3            192.168.56.104

实验结果包括：

- Ubuntu Server 安装成功
- NAT 网络可以访问外网
- Host-Only 网络可以实现宿主机与虚拟机互通
- Windows 可以通过 SSH 登录各虚拟机
- node1、node2、node3 主机名和静态 IP 配置成功
- /etc/hosts 主机名解析生效
- 多台虚拟机之间可以互相通信

## 六、遇到的问题与解决方法

### 1. 重启后再次进入 Ubuntu 安装界面

安装完成后，虚拟机重启时再次进入 Ubuntu 安装界面。原因是 Ubuntu ISO 安装镜像仍然挂载在虚拟光驱中，虚拟机优先从安装盘启动。

解决方法是在 VirtualBox 的存储设置中移除 Ubuntu ISO，使虚拟机从虚拟硬盘启动。

### 2. ping template 解析到 127.0.1.1

在配置 hosts 后，执行 ping template 时发现 template 被解析为 127.0.1.1。

原因是 /etc/hosts 中原本存在 127.0.1.1 template 这一行，系统优先使用了前面的匹配项。

解决方法是将该行注释掉，并添加 192.168.56.101 template。

### 3. 克隆机 SSH 主机身份重复

node1、node2、node3 从模板机克隆后，SSH Host Key 与模板机相同，Windows 连接时提示同一主机身份对应多个地址。

解决方法是在每台克隆机中删除旧的 SSH Host Key，并重新生成：

    sudo rm -f /etc/ssh/ssh_host_*
    sudo dpkg-reconfigure openssh-server
    sudo systemctl restart ssh

### 4. Git Bash/PowerShell 中 known_hosts 记录需要更新

重新生成 SSH Host Key 后，Windows 本地仍然可能保存旧的主机指纹。

解决方法是在 Windows PowerShell 中执行：

    ssh-keygen -R 对应IP地址

然后重新连接并确认新的主机指纹。

## 七、实验总结

通过本实验，我理解了虚拟机、宿主机、NAT、Host-Only 网络、静态 IP、主机名和 SSH 远程登录之间的关系。

VirtualBox 用于模拟多台服务器，Ubuntu Server 提供 Linux 服务器环境，NAT 负责虚拟机访问外网，Host-Only 负责宿主机和虚拟机之间通信，静态 IP 和 /etc/hosts 则让多节点环境更加稳定。

本实验为后续 Linux 环境配置、SSH 远程管理和网站服务器部署打下了基础。
